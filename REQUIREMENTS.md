## 1. 认证授权模块

### 1.1 功能概述

基于 Spring Security + JWT 的无状态认证系统，配合 RBAC 五角色权限模型，实现用户登录、Token 管理、菜单动态过滤和按钮级权限控制。

### 1.2 涉及角色

| 角色 | 角色编码 | 职责 |
|---|---|---|
| 超级管理员 | admin | 全部权限，可跨级审批，可导出全量数据 |
| 普通员工 | staff | 查看个人数据，提交请假/加班/报销/会议申请 |
| 部门经理 | manager | 员工权限 + 审批请假/加班/报销（一级审批） |
| HR管理员 | hr | 员工档案维护，公告发布与管理 |
| 财务管理员 | finance | 报销单终审（二级审批） |

### 1.3 功能需求

| 编号 | 需求描述 | 验收标准 |
|---|---|---|
| AUTH-01 | 用户使用用户名和密码登录 | 验证 BCrypt 密码，返回 JWT Token + 用户信息 + 菜单树 + 权限列表 |
| AUTH-02 | 账号状态校验 | 禁用账号（status≠1）登录时返回"账号已被禁用" |
| AUTH-03 | JWT Token 生成 | Token 包含 userId、username、roles，HMAC-SHA 签名，24 小时过期 |
| AUTH-04 | 后续请求自动认证 | JwtAuthenticationFilter 从 Authorization 头提取 Bearer Token，设置 SecurityContext |
| AUTH-05 | 获取当前用户信息 | GET /api/auth/me 返回用户信息 + 菜单 + 权限 |
| AUTH-06 | 退出登录 | 清除 SecurityContext，前端清除 localStorage |
| AUTH-07 | 菜单动态过滤 | 管理员返回全部菜单，其他用户通过 RBAC 四表联查返回有权限的菜单树 |
| AUTH-08 | 按钮级权限查询 | 返回 type=2 的权限编码列表（29 项），管理员返回全部 |
| AUTH-09 | 前端权限检查 | auth.js 提供 hasPermission()、canAccessMenu()、hasRole() 方法 |
| AUTH-10 | 认证异常处理 | 401 返回 JSON `{"code":401,"msg":"未登录或Token已过期"}` |
| AUTH-11 | 授权异常处理 | 403 返回 JSON `{"code":403,"msg":"权限不足"}` |

### 1.4 业务规则

- 密码以 BCrypt 哈希存储，不可逆，不可解密
- 登录时手动验证密码（`passwordEncoder.matches`），不使用 AuthenticationManager
- 前端 apiFetch() 全局拦截 `/api/` 请求自动携带 Token
- 管理员角色拥有所有权限，不需要查询 RBAC 关联表
- 角色编码在 SecurityContext 中以 `ROLE_` 前缀存储
- 会话无状态（SessionCreationPolicy.STATELESS），禁用 CSRF

## 2. 员工管理模块

### 2.1 功能概述

维护员工档案信息，支持分页查询和详情查看，关联展示员工考勤统计数据。

### 2.2 涉及角色

| 角色 | 权限 |
|---|---|
| admin | 查看全部员工列表，可增删改 |
| hr | 查看全部员工列表，可增删改 |
| manager | 查看员工列表 |
| staff | 查看员工列表 |

### 2.3 功能需求

| 编号 | 需求描述 | 验收标准 |
|---|---|---|
| EMP-01 | 员工列表分页查询 | 支持按关键词搜索，返回分页数据含部门名称 |
| EMP-02 | 员工详情查看 | 返回员工基本信息 + 部门 + 本月出勤天数 + 请假天数 + 加班小时数 |

- 删除员工时需校验是否有关联用户账号

## 3. 考勤管理模块

### 3.1 功能概述

以日历视图展示每日打卡记录，支持月度考勤统计，识别考勤异常（迟到、早退、缺卡）。

### 3.2 涉及角色

| 角色 | 权限 |
|---|---|
| admin | 查看全部考勤数据 |
| manager | 查看全部考勤数据 |
| staff | 查看自己的考勤数据 |

### 3.3 功能需求

| 编号 | 需求描述 | 验收标准 |
|---|---|---|
| ATT-01 | 考勤日历视图 | 返回指定月份每日打卡记录（签到时间、签退时间、状态），以日历网格展示 |
| ATT-02 | 月度考勤统计 | 返回正常出勤天数、迟到次数、早退次数、缺卡次数、加班天数 |
| ATT-03 | 考勤状态标识 | 每日期状态：正常/迟到/早退/缺卡/请假/加班/周末 |

- 日历接口支持年月参数，默认当月

## 4. 请假管理模块

### 4.1 功能概述

员工提交请假申请，支持 7 种请假类型，提交后进入单级审批流（部门经理审批）。

### 4.2 涉及角色

| 角色 | 权限 |
|---|---|
| staff | 提交请假申请，查看自己的请假记录 |
| manager | 提交请假申请，查看自己的请假记录，审批下属请假 |
| admin | 全部权限 |

### 4.3 功能需求

| 编号 | 需求描述 | 验收标准 |
|---|---|---|
| LV-01 | 请假申请提交 | 填写请假类型、开始日期、结束日期、天数、原因，系统自动生成请假单号 |
| LV-02 | 请假类型选择 | 支持病假、事假、年假、调休、婚假、产假、其他共 7 种类型 |
| LV-03 | 请假天数自动计算 | 根据开始日期和结束日期自动计算天数 |
| LV-04 | 审批状态流转 | PENDING → 经理审批 → COMPLETED 或 REJECTED |

### 4.4 业务规则

- 请假单号格式由系统自动生成，确保唯一性
- 提交后初始状态为 PENDING，进入审批流
- 单级审批：经理或管理员可直接将 PENDING 审批至 COMPLETED
- 驳回后状态为 REJECTED，不支持重新提交（需新建申请）

## 6. 费用报销模块

### 6.1 功能概述

员工提交费用报销申请，支持发票图片上传、费用明细行拆分，提交后进入两级审批流（经理→财务）。

### 6.2 涉及角色

| 角色 | 权限 |
|---|---|
| staff | 提交报销单，查看自己的报销记录 |
| manager | 提交报销单，一级审批（PENDING→MANAGER_APPROVED） |
| finance | 二级审批（MANAGER_APPROVED→COMPLETED） |
| admin | 全部权限 |

### 6.3 功能需求

| 编号 | 需求描述 | 验收标准 |
|---|---|---|
| ER-01 | 报销单创建 | 填写报销类型、总金额、描述，上传发票图片，可添加多条费用明细行 |
| ER-02 | 发票图片上传 | 支持 multipart/form-data 上传，文件存储在 uploads/invoice/ 目录，通过 /uploads/** 路径访问 |
| ER-03 | 费用明细行 | 每条明细行包含：费用项名称、金额、发生日期、备注 |
| ER-04 | 报销单号自动生成 | 系统自动生成唯一报销单号 |
| ER-05 | 我的报销单列表 | 分页展示当前用户提交的报销单，含审批状态 |
| ER-06 | 审批列表查看 | 按角色展示待审批的报销单 |
| ER-07 | 报销单详情 | 返回报销单头信息 + 费用明细行列表 + 发票图片 |
| ER-08 | 报销单状态更新 | 有权限的用户可通过审批流更新状态 |
| ER-09 | 报销单删除 | 仅创建人或管理员可删除 |

### 6.4 业务规则

- 两级审批链：PENDING → 经理审批 → MANAGER_APPROVED → 财务审批 → COMPLETED
- 任一节点可驳回至 REJECTED
- 发票上传限制大小 10MB（Spring Boot 默认 multipart 配置）
- 报销单与明细行是一对多关系，明细行随报销单级联操作
- 总金额由明细行金额汇总或手动填写

## 7. 审批流模块

### 7.1 功能概述

统一审批引擎，支持多业务类型（请假/加班/报销）的多级审批状态机，基于角色路由审批权限，提供审批时间线和待审批列表。

### 7.2 涉及角色

| 角色 | 审批权限 |
|---|---|
| admin | 全部审批权限，可跨级操作（PENDING 直接到 COMPLETED） |
| manager | 审批所有 PENDING 状态的单据（一级审批） |
| finance | 仅审批 MANAGER_APPROVED 状态的报销单（二级审批） |
| staff | 无审批权限 |
| hr | 无审批权限 |

### 7.3 功能需求

| 编号 | 需求描述 | 验收标准 |
|---|---|---|
| APR-01 | 审批执行 | 审批人通过或驳回申请，更新业务单据状态，记录审批记录 |
| APR-02 | 状态机流转 | 报销单两级审批（PENDING→MANAGER_APPROVED→COMPLETED），请假/加班单级审批（PENDING→COMPLETED） |
| APR-03 | 驳回处理 | 任意状态可直接驳回至 REJECTED，不限当前状态和审批人角色 |
| APR-04 | 管理员跨级操作 | admin 角色可将 PENDING 直接审批至 COMPLETED，跳过中间节点 |
| APR-05 | 审批时间线 | 按业务类型和业务ID查询完整审批链路（审批人姓名、结果、意见、时间） |
| APR-06 | 待审批列表 | 按角色过滤：manager 看所有 PENDING，finance 看 MANAGER_APPROVED 的报销单，admin 看全部 |
| APR-07 | 用户角色查询 | 查询用户当前拥有的角色编码列表 |
| APR-08 | 非法审批拦截 | 当审批人不具备当前状态的操作权限时，抛出异常并提示 |

### 7.4 业务规则

- 状态机核心代码位于 `ApprovalServiceImpl.computeNextStatus()`，根据 businessType + currentStatus + role 计算下一状态
- 审批记录写入 `approval_record` 表，不可修改，不可删除
- 审批意见为可选字段
- 驳回和通过均记录审批人和时间
- 事务保证：状态更新 + 审批记录插入在同一事务中

## 9. 会议预约模块

### 9.1 功能概述

员工预约会议室，系统自动检测时间冲突，生成预约单号，支持查看预约列表和取消预约。

### 9.2 涉及角色

| 角色 | 权限 |
|---|---|
| staff | 预约会议室，查看自己的预约 |
| manager | 预约会议室，查看自己的预约，可取消预约 |
| admin | 全部权限 |

### 9.3 功能需求

| 编号 | 需求描述 | 验收标准 |
|---|---|---|
| MEET-01 | 会议室预约 | 选择会议室、会议标题、开始时间、结束时间、描述，系统检测时间冲突并提示 |
| MEET-02 | 时间冲突检测 | 同一会议室在指定时间段内已有预约时，提示冲突不允许创建 |
| MEET-03 | 预约单号自动生成 | 系统生成唯一预约单号 |
| MEET-04 | 我的预约列表 | 查看当前用户的所有预约记录 |
| MEET-05 | 会议室日程查询 | 按会议室查看指定日期范围的所有预约 |
| MEET-06 | 取消预约 | 取消未开始的预约 |
| MEET-07 | 会议提醒 | 预约成功后自动加入 Redis ZSet 延迟队列，会议开始前 15 分钟发送通知 |

### 9.4 业务规则

- 冲突检测算法：同一会议室 + 时间段重叠（startTime < existingEnd AND endTime > existingStart）
- 预约取消后提醒任务也从 Redis ZSet 中移除
- 预约单号格式由系统自动生成
- 会议提醒通过定时任务（每 30 秒扫描 Redis ZSet）触发

## 10. 公告通知模块

### 10.1 功能概述

管理公司公告的发布、编辑、撤回，支持已读/未读状态追踪和未读数量统计。

### 10.2 涉及角色

| 角色 | 权限 |
|---|---|
| admin | 全部权限 |
| hr | 创建/编辑/发布/撤回公告 |
| manager | 查看已发布公告 |
| staff | 查看已发布公告 |

### 10.3 功能需求

| 编号 | 需求描述 | 验收标准 |
|---|---|---|
| NC-01 | 公告创建 | 填写标题、内容、类型（通知/公告），初始状态为 DRAFT |
| NC-02 | 公告编辑 | 修改未发布的公告内容 |
| NC-03 | 公告发布 | 状态从 DRAFT 变为 PUBLISHED，记录发布时间和发布人 |
| NC-04 | 公告撤回 | 状态从 PUBLISHED 变为 WITHDRAWN，已撤回的公告不再在列表中展示 |
| NC-05 | 已发布公告列表 | 所有用户可查看 PUBLISHED 状态的公告，按发布时间倒序 |
| NC-06 | 公告详情 | 查看公告完整内容 |
| NC-07 | 标记已读 | 用户点击公告后记录已读状态（sys_notice_read 表） |
| NC-08 | 未读数量统计 | 统计当前用户未读的 PUBLISHED 公告数量 |
| NC-09 | 已读状态查询 | 返回指定公告的已读用户列表和未读用户列表 |

### 10.4 业务规则

- 公告状态：DRAFT(0) → PUBLISHED(1) / WITHDRAWN(2)
- 撤回不可逆，需重新创建
- 已读记录与用户绑定，用于前端红点提示
- 公告类型：通知(1) / 公告(2)

## 12. 消息通知模块

### 12.1 功能概述

站内消息通知系统，支持审批结果通知、会议提醒、公告通知推送。基于 Redis ZSet 延迟队列实现定时提醒，定时任务每 30 秒扫描一次。

### 12.2 涉及角色

| 角色 | 权限 |
|---|---|
| 所有角色 | 查看自己的通知，标记已读 |

### 12.3 功能需求

| 编号 | 需求描述 | 验收标准 |
|---|---|---|
| NF-01 | 通知列表查询 | 返回当前用户的消息通知列表，按时间倒序 |
| NF-02 | 未读数量统计 | 统计当前用户未读通知数，支持红点提示 |
| NF-03 | 单条标记已读 | 用户阅读通知后标记为已读 |
| NF-04 | 批量标记已读 | 支持按通知ID列表批量标记已读 |
| NF-05 | 全部标记已读 | 一键将所有未读通知标记为已读 |
| NF-06 | 审批通知自动生成 | 审批通过/驳回后自动向申请人发送通知 |
| NF-07 | 会议提醒自动推送 | 会议开始前 15 分钟自动向预约人发送提醒通知 |
| NF-08 | 公告通知自动推送 | 公告发布后自动向相关用户发送通知 |
| NF-09 | 提醒延时队列 | 基于 Redis ZSet 的延时任务管理，score 为触发时间戳 |

### 12.4 业务规则

- 通知存储在 `oa_notification` 表
- 通知状态：UNREAD / READ
- Redis ZSet key 格式由 `RedisKeys` 常量类统一管理
- 定时任务 `ReminderScheduler` 每 30 秒扫描 ZSet 中到期的任务
- 到期任务生成 Notification 记录后从 ZSet 中移除
- BusinessType 枚举管理通知来源：ANNOUNCEMENT / MEETING / APPROVAL / LEAVE / OVERTIME / REIMBURSEMENT
